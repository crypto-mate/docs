This quickstart walks you through the minimum steps to authenticate against the CryptoMate API and make your first call from the **Sandbox** environment.

## 1. Create an account

Sign up on the [CryptoMate Dashboard](https://portal.cryptomate.me) and complete the onboarding for your organization. Once you're in, you'll have access to both environments:

- **Sandbox** — safe playground, no real-world blockchain impact. Use it for development and testing.
- **Production** — live transactions recorded on the main blockchain.

<Tip>
  Always build and test your integration in Sandbox before switching to Production.
</Tip>

## 2. Generate an API key

From the Dashboard, open the **API Keys** section and create a new key.

When creating the key you can:

- Scope its **permissions** to only the products and actions your integration needs
- Restrict it to a list of **allowed IPs** (recommended for backend services)

Copy the key immediately and store it in a secure location (environment variable, secret manager, etc.). You won't be able to see it again.

<Warning>
  Never commit API keys to source control or embed them in client-side code.
</Warning>

For a deeper dive, see [API Keys & Access Security](/authentication).

## 3. Make your first request

Include your API key in the `X-API-KEY` header of every request.

<CodeGroup>

```bash cURL
curl https://api.cryptomate.me/v1/blockchains \
  -H "X-API-KEY: $CRYPTOMATE_API_KEY"
```

```javascript Node.js
const res = await fetch("https://api.cryptomate.me/v1/blockchains", {
  headers: {
    "X-API-KEY": process.env.CRYPTOMATE_API_KEY,
  },
});

const blockchains = await res.json();
console.log(blockchains);
```

```python Python
import os
import requests

res = requests.get(
    "https://api.cryptomate.me/v1/blockchains",
    headers={"X-API-KEY": os.environ["CRYPTOMATE_API_KEY"]},
)

print(res.json())
```

</CodeGroup>

A successful response returns a `2xx` status code and a JSON body. If something goes wrong, inspect the `code` and `message` fields and note the `X-Trace-Id` header — see [Errors](/integration/errors).

## 4. Configure a webhook (optional)

Most production integrations need to react to events asynchronously (a deposit arrives, a card transaction is authorized, a withdrawal completes). Expose an HTTPS endpoint and register it from the Dashboard to start receiving events.

See [Webhooks](/integration/webhooks) for the payload format and the list of available events.

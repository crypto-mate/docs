---
title: "Create PIX bank account"
description: "Register a Brazilian PIX bank account for a client."
api: "POST /management/clients/{clientId}/banks/pix"
---

Creates a new Brazilian PIX bank account for the specified client. Requires an API key with access level 3 or higher.

## Path parameters

<ParamField path="clientId" type="string" required>
  Unique identifier of the client.
</ParamField>

## Body

<ParamField body="bank_name" type="string">
  Name of the bank.
</ParamField>

<ParamField body="account_alias" type="string">
  User-friendly alias for the account.
</ParamField>

<ParamField body="bank_holder_first_name" type="string">
  Account holder's first name.
</ParamField>

<ParamField body="bank_holder_last_name" type="string">
  Account holder's last name.
</ParamField>

<ParamField body="pix_key" type="string">
  PIX key (email, phone, CPF, or random key).
</ParamField>

<ParamField body="document_number" type="string">
  CPF or CNPJ document number.
</ParamField>

<ParamField body="country" type="string">
  ISO country code.
</ParamField>

<ParamField body="address" type="object">
  Account holder's address.
</ParamField>

## Response

See [Create US bank account](/api-reference/management/clients/bank-accounts/create-us-bank-account) for the full field list. PIX accounts expose `pix_key` and `document_number`.

<RequestExample>

```bash cURL
curl -X POST "https://api.cryptomate.me/management/clients/client_a1b2c3d4-e5f6-4a7b-8c9d-0e1f2a3b4c5d/banks/pix" \
  -H "x-api-key: $CRYPTOMATE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "bank_name": "Banco do Brasil",
    "account_alias": "Primary PIX Account",
    "bank_holder_first_name": "João",
    "bank_holder_last_name": "Silva",
    "pix_key": "customer@example.com",
    "document_number": "12345678900",
    "country": "BR",
    "address": {
      "streetLine1": "Av. Paulista 1000",
      "city": "São Paulo",
      "postal_code": "01310-100",
      "country": "BR"
    }
  }'
```

</RequestExample>

<ResponseExample>

```json 200 OK
{
  "id": "ba_jkl012",
  "client_id": "client_a1b2c3d4-e5f6-4a7b-8c9d-0e1f2a3b4c5d",
  "bank_name": "Banco do Brasil",
  "type": "PIX",
  "currency": "BRL",
  "pix_key": "customer@example.com",
  "document_number": "12345678900"
}
```

```json 404 Not Found
{
  "code": "NOT_FOUND",
  "message": "Client does not have a customer ID for provider BRIDGE"
}
```

</ResponseExample>
